[Heading 1] 1 · The 3-Level Display Pattern
[] APEX follows a three-level navigation pattern inspired by the best enterprise platforms of 2026. Each level uses a distinct UI affordance matched to cognitive load and usage frequency.
[Heading 3] Why Tabs + Dropdown (not pure slides, not pure dropdown)
[List Paragraph] Discoverability — Top 3–5 most-used screens visible at all times — users see them, can count them.
[List Paragraph] Scalability — When a sub-module has 8–12 screens, the less-used half goes into "More ▾" dropdown — no tab-row wrapping or squeezing.
[List Paragraph] Keyboard efficiency — Number keys 1–5 jump to visible tabs; ⌘K opens the dropdown searchable.
[List Paragraph] Mobile-ready — Tabs collapse to horizontal scroll on narrow screens; dropdown stays functional.
[List Paragraph] Permission-aware — Tabs the user cannot access are hidden — not grayed — to reduce cognitive clutter.
[Heading 3] Visual Behaviour
[List Paragraph] Active tab: brand-colored underline + bold label.
[List Paragraph] Hover tab: background tint + cursor pointer.
[List Paragraph] "More ▾" opens a popover listing the overflow screens with icons + short descriptions.
[List Paragraph] Popover supports search (⌘K inside), recent items pinned top, keyboard ↑↓.
[List Paragraph] User can pin an overflow item to visible tabs via right-click → "Pin to tabs".
[List Paragraph] Tab order is user-customizable (drag-to-reorder) and stored per user per sub-module.

[Heading 3] Layout Recap
[List Paragraph] Level 1 — Cards on Launchpad (6 colored cards + module switcher popover).
[List Paragraph] Level 2 — Sidebar items (46 sub-modules total, 6-11 per group, user-favorite pinning).
[List Paragraph] Level 3 — Top Tabs (up to 5 visible) + "More ▾" dropdown for overflow.
[List Paragraph] Shortcut: Alt+1..9 jumps to Nth sub-module · 1..5 jumps to Nth tab · ⌘K searches tabs.
[Heading 3] Customization Promise
[] Every user can pin any dropdown screen to visible tabs, reorder visible tabs, and save tab order per sub-module. Admins set tenant-wide defaults. Permissions hide unauthorized tabs entirely — no grayed items.

[Heading 3] Routing (GoRouter)
[List Paragraph] /app/{group}/{subModule}/{screen}
[List Paragraph] Examples: /app/erp/sales/invoices · /app/audit/workpapers/tick-marks · /app/feas/valuation/monte-carlo
[List Paragraph] Deep-linkable: pasting URL into another user's browser opens same screen (with permission check).
[Heading 3] Permission Matrix Tuning
[List Paragraph] RBAC capability format: {group}.{subModule}.{screen}.{action} — e.g., erp.sales.invoices.post, audit.workpapers.ticks.edit.
[List Paragraph] Role bundles pre-package common capability sets (Accountant, HR Manager, Auditor).
[List Paragraph] Field-level overrides: mark-up fields (salary, cost) to require additional capability.
[Heading 3] Analytics Events
[List Paragraph] module_switched, sub_module_opened, tab_activated, dropdown_opened, overflow_item_pinned.
[List Paragraph] Use these to auto-surface frequently used dropdown items ("Would you like to pin X?" suggestion after 3 opens in a week).
[Heading 3] First-Run Onboarding
[List Paragraph] New user lands on Launchpad. Coach-marks highlight the 6 module cards.
[List Paragraph] Click any card: coach-mark highlights sidebar, then tab bar, then "More ▾".
[List Paragraph] Shortcut card on first login ("Press ⌘K to search anything").
